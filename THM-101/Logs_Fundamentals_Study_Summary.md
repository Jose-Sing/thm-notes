## 1. What are Logs & Why They Matter

Logs are records of activities generated by every process running on a device or application — both interactive (user-initiated) and non-interactive (background). They are the **traces of everything that happens** on a system.

### Key Use Cases

| Use Case                               | Description                                                                                       |
| -------------------------------------- | ------------------------------------------------------------------------------------------------- |
| **Security Events Monitoring**         | Detect anomalous behavior through real-time log monitoring.                                       |
| **Incident Investigation & Forensics** | Provide a detailed trail of what happened during an incident — essential for root cause analysis. |
| **Troubleshooting**                    | Record system and application errors that help diagnose and fix issues.                           |
| **Performance Monitoring**             | Provide insights into the performance of applications over time.                                  |
| **Auditing & Compliance**              | Establish an activity trail that satisfies compliance requirements and supports audits.           |

> 💡 Manually searching for specific activity or anomalies across large log files is not feasible — manual and automated **log analysis techniques** are used instead.

## 2. Types of Logs

Logs are segregated into categories based on the type of information they contain. This allows investigators to go directly to the relevant log file instead of sifting through everything.

| Log Type             | Purpose                                                                                        | Examples of Events                                                                                            |
| -------------------- | ---------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **System Logs**      | OS-level activities — useful for troubleshooting running issues.                               | System startup/shutdown, driver loading, hardware events, system errors                                       |
| **Security Logs**    | Security-related activities — critical for incident detection and investigation.               | Authentication events, authorization events, security policy changes, user account changes, abnormal activity |
| **Application Logs** | Application-specific events — any activity inside an application.                              | User interactions, application changes, updates, application errors                                           |
| **Audit Logs**       | Detailed system changes and user events — important for compliance and security monitoring.    | Data access, system changes, user activity, policy enforcement                                                |
| **Network Logs**     | Incoming and outgoing network traffic — useful for troubleshooting and incident investigation. | Incoming/outgoing traffic, network connections, firewall logs                                                 |
| **Access Logs**      | Detailed records of access to different resources.                                             | Web server access, database access, application access, API access                                            |

> 💡 Additional log types exist depending on the application or service in use.

## 3. Windows Event Logs Analysis

Windows OS stores logs in segregated files, each covering a specific category. The three most important for security work are:

| Log File        | Content                                                                          | Security Relevance |
| --------------- | -------------------------------------------------------------------------------- | ------------------ |
| **Application** | Events from all running applications — errors, warnings, compatibility issues.   | Medium             |
| **System**      | OS operations — driver issues, hardware events, startup/shutdown, services.      | Medium             |
| **Security**    | All security-related activity — authentication, account changes, policy changes. | **Critical**       |

### Event Viewer

Windows provides a built-in GUI tool called **Event Viewer** to view, search, and filter logs.

**How to open:**

```
Start → type "Event Viewer" → Enter
```

**Navigation:**

- `Windows Logs` → shows the different log file categories
- Clicking a log file displays all its events
- Double-clicking an event shows its full details

### Windows Event Log Fields

Each event in Event Viewer contains the following key fields:

| Field           | Description                                                                         |
| --------------- | ----------------------------------------------------------------------------------- |
| **Log Name**    | The name of the log file the event belongs to (Application, System, Security, etc.) |
| **Event ID**    | A unique numeric identifier for a specific type of activity                         |
| **Logged**      | Timestamp — the exact date and time the event occurred                              |
| **Description** | Detailed information about the activity                                             |

### Filtering Events — Filter Current Log

Event Viewer's **Filter Current Log** feature allows searching by Event ID — essential for quickly finding specific activities without scrolling through thousands of entries.

```
Event Viewer → select log file → Filter Current Log → enter Event ID → OK
```

**Example:** Filter by Event ID `4624` to see all successful login events.

## 4. Important Windows Event IDs

Event IDs are unique identifiers for specific activities. Knowing the key ones is essential for efficient log analysis during investigations.

| Event ID | Description                                                |
| -------- | ---------------------------------------------------------- |
| `4624`   | A user account **successfully logged in**                  |
| `4625`   | A user account **failed to log in**                        |
| `4634`   | A user account **successfully logged off**                 |
| `4720`   | A user account was **created** |
| `4722`   | A user account was **enabled**                             |
| `4724`   | An attempt was made to **reset an account's password**     |
| `4725`   | A user account was **disabled**                            |
| `4726`   | A user account was **deleted**                             |

> 💡 Many more Event IDs exist. Memorizing all of them is not necessary, but knowing the most critical ones (especially `4624` and `4625`) speeds up security investigations significantly.

## 5. Web Server Access Logs Analysis

Every request made to a website is logged by the web server. These logs are invaluable for detecting attacks, investigating incidents, and understanding traffic patterns.

**Default Apache access log location:**

```
/var/log/apache2/access.log
```

### Access Log Fields

Each log entry contains the following fields:

| Field           | Example                  | Description                                                  |
| --------------- | ------------------------ | ------------------------------------------------------------ |
| **IP Address**  | `172.16.0.1`             | IP of the client that made the request                       |
| **Timestamp**   | `[06/Jun/2024:13:58:44]` | Date and time the request was received                       |
| **HTTP Method** | `GET`                    | The action requested (GET, POST, PUT, DELETE, etc.)          |
| **URL**         | `/products`              | The resource being requested                                 |
| **Status Code** | `200`                    | Server's response — indicates success, error, redirect, etc. |
| **User-Agent**  | `Mozilla/5.0 (...)`      | Client OS, browser, and version information                  |

### Sample Log Entry

```
172.16.0.1 - - [06/Jun/2024:13:58:44] "GET /products HTTP/1.1" 404 "-" "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
```

Broken down:

- IP: `172.16.0.1`
- Time: `06/Jun/2024 at 13:58:44`
- Method: `GET`
- Resource: `/products`
- Status: `404` (Not Found)
- Browser: Chrome on Windows 10

## 6. Linux CLI Tools for Log Analysis

Manual log analysis on Linux is done using standard CLI utilities. These are essential for working with web server access logs and any text-based log file.

### `cat` — Display log file contents

```bash
# Display entire log file
cat access.log

# Combine multiple log files into one
cat access1.log access2.log > combined_access.log
```

> 💡 Log rotation is a common practice — systems split logs into separate files per timeframe. `cat` can merge them back into a single file for analysis.

---

### `grep` — Search for strings or patterns

```bash
# Find all entries from a specific IP
grep "192.168.1.1" access.log

# Find all POST requests
grep "POST" access.log

# Find all 404 errors
grep " 404 " access.log
```

**Example output:**

```
192.168.1.1 - - [06/Jun/2024:13:56:44] "GET /about HTTP/1.1" 500 "-" "Mozilla/5.0..."
192.168.1.1 - - [06/Jun/2024:13:53:44] "GET /products HTTP/1.1" 404 "-" "Mozilla/5.0..."
192.168.1.1 - - [06/Jun/2024:13:46:44] "GET /about HTTP/1.1" 200 "-" "Mozilla/5.0..."
```

---

### `less` — View large logs page by page

```bash
less access.log
```

| Key        | Action                                        |
| ---------- | --------------------------------------------- |
| `Space`    | Move to the next page                         |
| `b`        | Move to the previous page                     |
| `/pattern` | Search for a pattern within the file          |
| `n`        | Jump to the next occurrence of the search     |
| `N`        | Jump to the previous occurrence of the search |
| `q`        | Quit                                          |

> 💡 `less` is the preferred tool when dealing with very large log files — it loads only what's visible instead of the entire file, making it much more efficient than `cat` for large datasets.

## 7. Quick Reference

### Log Type → Investigation Scenario

| Investigation Need                  | Log to Check                                           |
| ----------------------------------- | ------------------------------------------------------ |
| Successful/failed logins on Windows | Security Logs → Event ID `4624` / `4625`               |
| User account created or deleted     | Security Logs → Event ID `4720` / `4726`               |
| System crash or driver issue        | System Logs                                            |
| Application throwing errors         | Application Logs                                       |
| Web requests to a server            | Web Server Access Logs (`/var/log/apache2/access.log`) |
| Network traffic anomalies           | Network Logs / Firewall Logs                           |
| Data access trail for compliance    | Audit Logs                                             |

### CLI Log Analysis Cheatsheet

```bash
# View full log
cat access.log

# Merge two log files
cat access1.log access2.log > combined.log

# Search for a specific IP
grep "192.168.1.1" access.log

# Search for all 500 errors
grep " 500 " access.log

# Search for POST requests
grep "POST" access.log

# Page through a large log
less access.log
# Then: / to search | n next | N previous | Space next page | b back | q quit
```

### Key Windows Event IDs at a Glance

```
4624 → Successful login
4625 → Failed login
4634 → Logoff
4720 → Account created
4722 → Account enabled
4724 → Password reset attempt
4725 → Account disabled
4726 → Account deleted
```

### Key Terminology

| Term             | Definition                                                                                                                 |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **Log**          | A recorded entry of an activity or event generated by a process, application, or system.                                   |
| **Log Analysis** | The technique of extracting valuable information from logs by identifying abnormal or suspicious activity.                 |
| **Event ID**     | A unique numeric identifier assigned to a specific type of activity in Windows event logs.                                 |
| **Event Viewer** | Windows built-in GUI tool for viewing, filtering, and searching Windows event logs.                                        |
| **Log Rotation** | The practice of splitting logs into separate files by timeframe to prevent a single log file from growing indefinitely.    |
| **User-Agent**   | A string in HTTP requests that identifies the client's OS, browser, and version.                                           |
| **Status Code**  | A 3-digit HTTP response code indicating the result of a web request (200 = OK, 404 = Not Found, 500 = Server Error, etc.). |
| **Access Log**   | A web server log recording all incoming HTTP requests with their IP, timestamp, method, URL, status code, and user-agent.  |
| **`cat`**        | Linux CLI utility to display or combine text file contents.                                                                |
| **`grep`**       | Linux CLI utility to search for strings or patterns within files.                                                          |
| **`less`**       | Linux CLI utility to view large files one page at a time with search capability.                                           |
